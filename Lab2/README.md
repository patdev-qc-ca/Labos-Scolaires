# Equipe4-Laboratoire2

# Mise en situation

Vous devrez créer une application web d’achats de vêtements. 
+ Chaque vêtement a un inventaire (qui devra être mis à jour lors des ajouts au panier) et peut être ajouté au panier de vente d’utilisateurs sous forme de « ligne de panier ». 
+ Le panier peut être accédé avec la requête /{utilisateur_id}/panier. 
+ De plus, lors que nous faisons une requête POST vers le endpoint /commandes, le panier de l’utilisateur fourni est vidé et le contenu est ajouté sous forme de « lignes de commande ». La gestion de commandes, la gestion de vêtements, ainsi que la gestion d’utilisateurs se font sous le endpoint /admin/{action}.
+ Pour la gestion de commandes, une commande peut être validée. Pour la gestion d’utilisateurs, un utilisateur peut être ajouté, mis à jour ou supprimé. 
+ Si un utilisateur est supprimé, toutes ses commandes et paniers devront l’être aussi. 
+ Pour la gestion de vêtements, ceux-ci peuvent être ajouté ou mis à jour.
